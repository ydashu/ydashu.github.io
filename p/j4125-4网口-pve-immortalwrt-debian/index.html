安装immortalwrt

  • 下载immortalwrt镜像 - 版本和架构自行选择
    • 下载generic-squashfs-combined-efi.qcow2 PVE属于KVM
      类型 推荐文件格式/后缀 适用场景
      实体机安装 .img.gz (combined) 软路由/物理设备刷机
      虚拟机专用 .vmdk (VMware) .qcow2 (KVM) 对应虚拟机平台直接加载
      UEFI支持 含efi的文件名 新型主板/UEFI启动环境
      BIOS支持 不含efi的文件名 传统Legacy启动模式
      稳定运行 squashfs后缀 只读系统，防配置错误
      可写系统 ext4后缀 需要频繁修改系统的开发者
      ISO启动 .iso 光盘/U盘启动安装
  • ftps连接PVE
  • 复制刚刚下载的文件到/var/lib/vz/template/iso/
  • 创建虚拟机

    • 操作系统：不适用任何介质

    • 系统

      • SCSI控制器：VirtIO SCSI single
        控制器类型 性能 兼容性 特点 推荐场景
        VirtIO SCSI ✅✅✅ 中等 高性能、支持多队列/热插拔 Linux/Windows（推荐主力使用）
        LSI SAS ✅ ✅✅ 兼容性好，适合无VirtIO驱动系统 Windows 7/10等通用系统
        VMware PVSCSI ✅✅ 中等 VMware专用控制器 VMware迁移系统
        VirtIO SCSI Single ✅✅ 中等 单磁盘独立控制器 OpenWRT等简单场景
        LSI 53C895A ❌ ✅✅✅ 兼容旧系统但性能差 Windows XP/2003等旧系统

    • 磁盘：删除不要（后面命令行安装immortalwrt时会自动创建一个磁盘）

    • CPU

      • 插槽：1
      • 核心：2

    • 内存：2048或1024

    • 网络

      • 模型：VirtIO(半虚拟化)
        模型名 类型 性能 兼容性 驱动需求 说明
        Intel E1000 仿真 ⭐⭐ ⭐⭐⭐⭐ 系统自带 模拟英特尔千兆网卡 82540EM，几乎所有系统都能识别。适合旧系统使用。性能较低。
        Intel E1000E 仿真 ⭐⭐ ⭐⭐⭐⭐ 系统自带 模拟较新 E1000e（如 I217LM）型号，稍快于 E1000。
        VirtIO (半虚拟化) 半虚拟化 ⭐⭐⭐⭐⭐ ⭐⭐ 需安装 VirtIO 驱动 最快、最轻量，占资源最少。推荐 Linux 系统或安装了 virtio 驱动的 Windows 系统使用。
        Realtek RTL8139 仿真 ⭐ ⭐⭐⭐⭐⭐ 系统自带 速度慢，仅 100Mbps，兼容性极强，适合非常旧的系统。
        VMware vmxnet3 半虚拟化 ⭐⭐⭐⭐ ⭐⭐ 需 VMware 驱动 VMware 专用，PVE/KVM 中不常用。性能好，但需要额外驱动。

    • 桥接：vmbr1

  • ssh登录PVE后台

        接口类型 特点描述 性能 启动兼容性 热插拔支持 推荐使用场景
        IDE 最老的虚拟化磁盘接口，兼容性最好，但速度慢 🚫 最差 ✅ 最兼容 ❌ 不支持 仅用于旧系统或特殊用途

        SCSI (VirtIO SCSI) 支持多磁盘、热插拔、性能略低于 VirtIO Block，功能更全面（推荐） 🟢 高 ✅ 支持 ✅ 完整支持 多磁盘 VM，现代 Linux，OpenWrt等

sshd_config 常用配置参数表
参数名 默认值 说明
Port 22 SSH 服务监听端口，可自定义为其他端口增强安全性
AddressFamily any 指定监听的地址族（inet：IPv4，inet6：IPv6，any：全部）
ListenAddress 0.0.0.0 指定监听的 IP 地址（默认所有网卡）
PermitRootLogin prohibit-password 是否允许 root 用户登录，常见值：yes、no、without-password
PasswordAuthentication yes 是否允许密码方式登录
PubkeyAuthentication yes 是否允许使用 SSH 公钥认证
ChallengeResponseAuthentication no 是否启用质询响应（如 OTP 认证）
UsePAM yes 启用 PAM（Pluggable Authentication Modules），控制登录策略
AllowUsers (未设置) 只允许指定用户连接（可多个，用空格分隔）
AllowGroups (未设置) 只允许指定用户组成员连接
DenyUsers (未设置) 拒绝指定用户连接
DenyGroups (未设置) 拒绝指定用户组成员连接
LoginGraceTime 120 用户认证超时时间（秒）
MaxAuthTries 6 最大认证失败次数，超过后断开连接
MaxSessions 10 每个连接允许的最大并发 session 数量（例如 shell、端口转发等）
PermitEmptyPasswords no 是否允许空密码用户登录（强烈建议保持为 no）
X11Forwarding yes 是否允许 X11 图形界面转发
PrintMotd yes 登录后是否显示 /etc/motd 消息
Banner (未设置) 登录前显示的横幅文件路径（如安全警告等）
LogLevel INFO 日志详细程度（如：QUIET、FATAL、ERROR、INFO、VERBOSE）
Subsystem sftp /usr/lib/openssh/sftp-server 指定 SFTP 子系统路径
