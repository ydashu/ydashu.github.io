clash

2025-04-09

发展历史

起源与诞生（2018-2019）

  • 开发者：Clash 由 Dreamacro（一位 GitHub 用户）于 2018 年发起，最初是一个基于 Go 语言开发的代理工具。
  • 目标：Clash 的设计初衷是提供一个支持多种代理协议的统一客户端，解决当时代理工具（如 Shadowsocks、V2Ray）功能分散、配置繁琐的问题。
  • 早期版本：
    • 2018 年底，Clash 项目在 GitHub 上发布，最初仅支持 Shadowsocks 协议。
    • 2019 年，陆续增加了对 V2Ray（VMess）、HTTP/HTTPS 等协议的支持。
  • 特色：Clash 引入了基于 YAML 的配置文件，支持灵活的分流规则（例如按域名或 GeoIP 路由），这在当时是创新性的设计。

快速成长与社区认可（2019-2020）

  • 功能扩展：
    • 2019 年中，Clash 添加了对 Trojan 协议的支持，进一步扩大了适用范围。
    • 引入 RESTful API（外部控制器）和 Web Dashboard，允许用户动态管理代理规则。
  • 跨平台支持：
    • Clash 被设计为跨平台工具，支持 Linux、Windows、macOS，甚至可以在 OpenWrt（路由器固件）上运行。
    • 社区开发了 Clash for Windows 和 ClashX（macOS）等第三方 GUI 客户端，大幅降低了使用门槛。
  • 社区活跃：
    • Clash 凭借其强大的分流功能和易扩展性，迅速在代理用户中流行，尤其受到中国大陆用户欢迎，用于科学上网。
    • GitHub 上 Star 数量快速增长，教程和配置文件分享社区（如 Clash 订阅服务）随之兴起。

成熟与巅峰（2020-2022）

  • 关键特性：
    • 2020 年，Clash 引入 TUN 模式（虚拟网卡），支持透明代理，使其从应用层代理升级为系统级解决方案。
    • 支持 GeoIP 和 Geosite 数据库，增强了分流精度。
  • 版本迭代：
    • 2021 年，Clash 发布 v1.0.0，标志着项目进入稳定阶段。
    • 到 2022 年，版本更新至 v1.18.0，修复了大量 Bug 并优化了性能。
  • 生态系统：
    • Clash Premium（闭源付费版）推出，提供高级功能（如更强的 TUN 支持和性能优化），但开源版本仍为主流。
    • 第三方工具（如 Clash for Android）进一步扩展了其生态。

停更与分裂（2023）

  • 停止更新：
    • 2023 年初，Dreamacro 宣布停止维护开源 Clash 项目（Dreamacro/clash），最后一个版本为 v1.18.0。
    • 具体原因未公开，可能与开发者个人精力、法律压力或商业化转向（如 Clash Premium）有关。
  • 社区反应：
    • Clash 的停更引发社区震动，但其代码开源（MIT 许可）为后续发展留下了空间。
    • 多个分支项目迅速涌现，试图接手维护和改进。

分支与新生（2023-至今）

  • Clash Premium：
    • Dreamacro 推出了闭源的 Clash Premium，作为官方付费替代品，增加了企业级功能，但未开源，社区接受度有限。
  • Clash.Meta / Mihomo：
    • 由 MetaCubeX 团队接手，基于 Clash 的一个分支（Clash.Meta）发展为 Mihomo。
    • 新增支持 Hysteria、TUIC 等新兴协议，优化性能，并提供 Mihomo Party（GUI 客户端）。
    • 截至 2025 年 4 月，Mihomo 仍在积极维护，成为开源 Clash 的主要继承者。
  • 其他分支：
    • Clash Verge：一个带 GUI 的客户端，整合 Clash 核心，适合桌面用户。
    • Clash Rust：尝试用 Rust 重写 Clash，但进展较慢。
  • 生态延续：
    • Clash 的配置文件格式（YAML）和分流逻辑已成为行业标准，被 Sing-box 等新工具兼容。

技术与文化影响

  • 技术贡献：
    • Clash 推动了代理工具的模块化和规则化设计，其分流机制被广泛模仿。
    • TUN 模式和外部控制 API 为后续工具（如 Mihomo、Sing-box）奠定了基础。
  • 文化意义：
    • Clash 成为科学上网的代名词，象征着开源社区对抗封锁的努力。
    • 其停更与分支发展也反映了开源项目生命周期的典型模式：从个人驱动到社区接力
